模板函数

  • ```c++
    #include
    using namespace std;

    //模板声明
    template
    T add(T a, T b) {

      return a + b;

    }

    int main() {

      cout << add(1, 2) << endl;     
  cout << add(1.5, 2.8) << endl;
  return 0;

    }

          -   任何类型都有默认构造函数 `T()`会构造一个临时变量，同样`U()`也会，这样decltype就可以知道返回什么类型了
      -   :smile:但是谁说就一定会有默认构造函数，可能有些自定义类型没有默认构造函数，那么同样的这样写也会报错，所以这种方法不可取

    • 解决③：返回值后置

      • C++11支持的

      •   auto func(int a, int b) -> decltype(a + b) {   //返回值后置
      return a + b;
  }

      • 这样就解决了上面所有问题

模板的偏特化和特化

  • 模板的偏特化：在某些特殊情况下，模板的行为方式有些不太一样

    • 可以看成相应模板的重载形式
    • 用来处理当前类型是指针或者数组这些情况的

  • 模板的特化：模板的性质都没有了，模板类型全都实例化完了，但是本质上还是一个模板

    • 可以理解为字面量，不可扩展的模板

  • 为什么模板的特化要加上template<>

    • 因为我们想要显示的调用模板是，希望能准确的匹配到这类模板，否则会匹配到其它形式的模板
