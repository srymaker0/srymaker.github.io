继承

  • 如上

    • Animal为父类/基类
    • Cat为子类/派生类
    • class Cat : public Animal {};
      • 这里的public表示继承权限，

  • 性质

    • 子类可以将父类中所有的成员属性和方法继承过来
    • :exclamation:拥有，不代表能访问

  • 继承权限

    • 继承权限不影响子类对父类内容的访问

      • 子类之于父类来说是类外部——在子类中不论用怎样的继承权限，都只能访问父类中的public和protected属性

    • protected：这个访问权限只提供给子类，只有子类才能访问protected权限的属性

  • 影响类外部访问子类内部继承自父类的成员属性的访问权限
    • 若继承权限是public，那么子类中继承自父类的属性的对外访问权限和父类中的成员属性对外访问权限一样。
    • 若继承权限是protected，那么继承自父类的属性访问权限原来是public的会被降级至protected，等于说类外就无法访问子类的属性，但是子类的子类却可以访问。
    • 若继承权限是private，那么谁都无法访问子类中继承自父类的属性。
    • 子类的地址可以转换成父类的地址——前提条件：子类必须公有继承自父类

继承-构造函数

  • 子类在进行构造的时候，先进行父类数据区的构造，然后再进行自己数据区的构造——子类会调用父类的构造函数
    • 子类中有一片数据区来自父类，当我们对子类的数据区去构造时，也要对父类的数据区去构造
  • 所以说父类先构造，子类后构造
    • 那么析构顺序是：子类先析构，父类后析构

注意拷贝和赋值

  • 会引起父类的存储区没有正确的完成拷贝动作
  • 必须显示的调用父类的拷贝构造和显示的调用赋值拷贝构造

菱形继承

  • 情况——A类派生出B、C类， D继承自B和C类

    • A、B、C类中都有run方法，那么请问D调用的是哪个类中的run方法？
    • 其实这是C++未定义的行为，具体调用B的还是C的不一定——这是bug

  • 在D的存储区中有B、C两部分的，而B、C两部分又包含了A

    • 所以当我修改了x的值其实是修改了B中继承自A的x值

  • 当我调用getX时其实是调用了C中未修改的x值——验证：输出x的地址，自行验证

    • 解决①：其他语言比如C#，java不允许多继承，只能继承一个基类，允许继承多接口

  • 解决②：利用虚继承——virtual关键字

      -   在有虚继承的类中，编译器会对这样的类做一点“手脚”
  -   ![](https://i.loli.net/2021/03/09/uKg5RaL7UsDWZMY.png)
  -   会把B和C中相同的部分给合并成一块空间

  • 在C++工程开发中，若想要使用菱形继承完成某项功能，则要基于

    • 继承一个真实基类，多个功能性基类

    • 真实基类：比如Animal类

    • 功能性基类：为了某个功能而创造出来的类

      • 功能需求：在一个项目要求某些类不能被拷贝

      • 【ps】当默认拷贝构造被删除时，默认构造函数也会被删除

      • 这个Uncopyable就是所谓的功能性的继承
