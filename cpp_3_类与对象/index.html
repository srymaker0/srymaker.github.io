类与对象

什么是类与对象

  • 类➡️类型， 对象➡️变量. 类型 = 类型数据 + 类型操作
    • 比如一个int类型，32位中占4字节，支持+-*/%操作 是不是很像 结构定义 + 结构操作
    • 认知提升：从思维逻辑层面来讲其实这些类型本质上也是一种数据结构
    • 也就是说，类是一种高级类型，对象是一种高级变量，

  • 成员属性可以理解为成员的数据

  • 成员方法可以理解为对成员数据的操作

    • 定义在类外和类内

    • 但是一般我们把成员方法定义在类外（一定要加作用域），把声明放在类内

类内和类外

  • People类里的都叫类内， 不在People类里的是类外

    • 类内的成员方法可以访问私有的成员变量， 而类外的一些函数无法访问类内的私有成员变量

  • class类的默认访问权限是private

命名空间

  • 是一个集合，一个限定域

  • namespace ZZZZZZ 👉表示定义一个ZZZZZZ的命名空间

    • ZZZZZZ表示的是一个命名空间

    • using namespace ZZZZZZ 表示使用ZZZZZZ命名空间下的东西

    • C++编码规范中不建议我们用using namespace ，引入的标识符太多了👇

    • 因为自己定义的变量和命名空间中的标识符可能会一样，这样会发生冲突

    • 提倡：用什么标识符就自己写上 比如代码11行

  • ⭐类是一个特殊的命名空间

    • 看People::say() 这表示的是People类下的say方法。跟命名空间一个道理

  • $this$指针——❗如图：this指针存储的是创建的对象的地址
    • 可以明确强调调用的是成员属性
    • 只有在成员方法内才能调用this指针
    • 指向的是当前对象

简版cout的实现

  • 首先需要理解cout是什么——cout是一个对象

    • 左移运算符是一个运算符，而运算符两边应该接变量，所以cout应该是一个变量
    • 但是在c++中高级变量可以叫对象，故cout是一个对象

  • 任何表达式都有返回值

    • 第36行，加括号这段代码是一个表达式，那么它返回值类型是什么呢？
    • cout是一个对象，n是一个变量，所以只有<<运算符有返回值，看26行，返回的是对象本身也就是cout
    • 这样就可以接收后面的字符串了
    • 24行为什么要加引用&？
      • 要想连续使用<<必须加&，因为要返回对象本身去接收后面的参数
    • 综上：cout是一个ostream类型的对象，cout << n这句话其实是cout这个对象利用重载运算符<<调用了函数输出n的

类属性与方法

类属性

  • 用static关键字进行修饰

  • 所有对象共有的一个属性

  • 在任何一个对象中访问一个类属性，访问到的都是同一个属性，类似与静态变量，全局唯一

  • 比如全人类的数量这个属性，并不是某个类独有的，它属于所有类

类方法

  • 不单独属于某个对象的方法
  • 不跟对象绑定，因此没有办法访问this指针

const方法——无敌属性

  • 对于一个const对象来说，是不允许改变对象内部的属性的，但是普通的成员方法的意思是可以改变成员属性的
  • 所以const方法就用来保证不会改变当前对象的所有属性
  • 一旦在const方法内部更改成员属性，会在编程器层面报错
class People {
public :
    .......
    void say() const {
        cout << "hahaha,funny!" << endl;
        say_cnt += 1;
        return ;
    }
    .......
private :
    int say_cnt;          //记录说话的次数
    .......
};

.......
  • 假设我在示例1中加这段代码，编译器肯定会报错
  • 因为我们正在一个const方法中尝试修改一个非const的值
    • 但是say_cnt用作于记录说话的次数，对于用户本身来说这个值的改变并没有影响，这就是逻辑意义上的const
  • 只要在say_cnt类型前加上mutable关键字修饰就可以了
  • mutable——免疫无敌：这个变量可变可不变，不归入数据意义上的管辖
  • 在const中如果还要调用方法，那么这个方法必须为const方法

  • 对象与引用

    • 就是一个变量的别名

    • ❗引用在定义的时候就要初始化

  • C++中的结构体与类

    • C++中的struct已经不是结构体了，struct在底层实现上是一个类

    • struct与class的区别：

      • struct的访问权限默认为public（黑名单机制，不想让外界访问某一个属性和方法时就要显示性的定义在private访问权限内）

      • class的访问权限默认为private（白名单机制，想要让外界访问某一个属性和方法时就要显示性的定义在public访问权限内）

返回值优化

  • 这里的理解：我们定义的一个a对象，将a对象作为参数传入func函数，而func函数的参数类型为SomeClass, 此时的初始化过程为👇
    • 先开辟b的对象数据区， 将a拷贝给b，调用拷贝构造函数，最后完成构造。

  • 通常情况下，函数的返回值会对应一个临时匿名对象

    • 当我们返回temp_a时相当于把temp_a赋值给了这个临时匿名对象，此时调用临时匿名对象的拷贝函数
    • 然后再调用a的拷贝构造函数
    • 整个过程调用了两次拷贝构造函数和一次有参构造

    • 这里模拟了整个过程，其中anonymous相当于匿名临时变量，因为其像一个中间商，所以编译器会执行第一次优化，减少一次拷贝构造
      • 第一次优化直接把anonymous给消灭了，让temp_a直接拷贝给a对象
      • 第二次优化：因为temp_a是局部变量，如果将来对temp_a做的所有操作都会加在a对象上的话，这时莫不如将temp_a看作是a的别名
        • 此时只调用一次有参构造，没有任何的拷贝行为👇

    • 编译器在调用拷贝构造函数是会进行优化
    • 在不同的环境中调用了几次拷贝构造函数你是不确定的
    • 所以在工程设计时不能改变拷贝构造函数语义——编译器可能不会知道你改了拷贝构造的语义

malloc与new

  • 可以看到我们在用malloc和new开辟内置类型的空间时效果是一样的，但是当开辟类类型的空间时，我们发现new可以在每个被开辟的空间中初始化A的对象，而malloc就只是开辟了一段空间而已
  • delete与free
    • delete可以在回收对象的时候，调用对象的析构函数，而free不行
  • ⭐malloc并不是一无是处，在实际应用中，其实是malloc加上new用原地构造的形式开辟空间。👇
    • 先用malloc开辟一段连续的存储空间（A 类型），然后再用new在开辟的每一位上依次进行构造
    • 42行代码的解释是：传入Adata1的首地址， 在它每一位上调用A的默认构造函数。
