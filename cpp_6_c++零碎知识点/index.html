c++零碎知识点

auto

  • 作用：在C++中可以自动推导变量的类型

    • 其实在C语言中就有，只不过在C语言中在变量类型前加auto表现的就像局部变量，多余
    • :exclamation:在编译期的行为

  • 限制

    • ①不能作为函数参数
    • ②不能作为模板参数（C++11）, 但是在C++14和C++17中可以作为模板参数
    • ③不能定义数组
    • ④不能定义非静态的成员属性

  • 容器

    • 封装好的数据结构——比如：map<int, int>

    • 遍历容器的工具类型：迭代器类型——map<int, int>::iterator👉遍历整型到整型的map类型的迭代器类型

    • iter->first:point_right:map中的key值， iter->second:point_right:map中的value值​

    • 但是这个迭代器类型太长了，所以auto关键字就起作用了，可以把第5行代码改成下面这种形式

  • 问题

    • ```c++
      auto y = 12.3; //编译器怎么推导出y到底是double类型还是float类型 //使用typeid运算符， 返回的是类型信息对象，若要获取类型信息的名字， 就像下面这样写
      cout << typeid(y).name() << endl; //输出一个d，表示double类型

constexpr——表达式常量

  • 区别于const
关键字 const constexpr
语义 不严谨 严谨
发生时段 运行期 编译期
定义的常量值 可以在运行期确定 必须在编译期就确定
  • 第10行为什么会输出123？
    • 因为编译器认为x是一个常量，作了优化，可以理解为直接从第2行代码跳到了第10行
    • 取地址是一个运行期的功能，不会被编译器优化，所以能改变x的值
  • constexpr所定义的变量，在编译期就要确定变量的值是多少。
  • 同样的y变量若是接受一个函数返回值，那么这个函数的返回值也必须修饰为constexpr，让编译器知道这是在编译期就能够确定的值
  • constexpr定义的对象调用的构造函数也要用constexpr修饰

nullptr

  • NULL和nullptr
    • NULL是一个整数0，同时也代表了空地址? 32行不知道调用哪个func
    • :star:nullptr只代表了空地址，更加明确，建议在C++中若想表示空地址，尽量用nullptr

左值与右值

  • 变量x是一个左值，绑定在左值上的引用就是左值引用

    • 语法：int y = 123; int &x = y; 此时x就是一个左值引用

  • 常量1234是一个右值， 绑定在右值上的引用就是右值引用

    • 语法 ：int &&x = 123; 此时x就是一个右值引用

  • :question:如何区别左值和右值

    • 当我们看到一个值时，到了代码的下一行的时候，能通过单一变量访问到相关的值，能->左值 不能->右值
    • 常量一定是一个右值
    • :exclamation:注意：左值，左值引用，右值，右值引用，这四个东西其实并不冲突，可以当成四种概念去学！

  • move和forward

    • 作用：都可以把一个具有左/右值特性的变量强制转换为右/左值特性
    • 区别：forward更加灵活，可以更改类型，forward<int &&> (x) 表示一定会匹配第一个参数是int &&的函数 👉 参数的完美转发
    • 准确的调用到函数的某种重载形式

  • 可以参考

    • C++：浅谈右值引用
    • 关于++i和i++的左值、右值问题

  • 左值右值更严谨的判断方式

  • 可以看到把func1函数的其它重载形式给注释了，发现调用的全是const int&版本:exclamation:

    • :star:可以说明的是const int &可以处理所有的数据

:star:移动构造

  • 作用：提高拷贝效率
    • 直接把数据拿过来，然后只需改动一下指针就可以
    • 配合深拷贝
  • 因为C++11标准引入了右值和右值引用的概念，这才有了移动构造，让C++重回神坛的原因
