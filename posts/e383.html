模板函数


#include <iostream>
using namespace std;
    
//模板声明
template<typename T>
T add(T a, T b) {
    return a + b;
}
    
int main() {
    cout << add(1, 2) << endl;     
    cout << add(1.5, 2.8) << endl;
    return 0;
}

  • 定义形式template<typename T>

    • typename 表示声明T是一个类型

    • 将参数类型给抽象化

    • 模板这段代码发生在编译期

    • 实例化：

      • 光有模板是不会实例化任何代码的，只有我们调用相关模板的时候才会实例化

      • 我们称这是模板声明，程序编译阶段其实模板已经实例化完了，在编译阶段之前，编译器就已经完成了模板的参数推导，实例化完后就没模板的事了👇

    • ❗在工程开发中，模板这段代码必须写在头文件中​——因为模版是一个声明
      • 可以使用nm -C命令 看编译代码

  • 问题1👉若是调用add(1, 1.5)这样的函数呢？ 上面这段代码不是炸了？——两个数据的类型不一样

    • 解决①：使用add<double>(1, 1.5)这样的调用形式， 这样可以让T类型强制变成double类型

    • 解决②：使用decltype关键字

      • decltype(1 + 2) x 表示定义了一个整型变量x， 其中decltype(1 + 2)是类型，decltype会自动推导括号里表达式的类型

template<typename T, typename U>
decltype(T() + U()) add(T a, U b) {
    return a + b;
}
//decltype(a + b) add(T a, U b) 这样的写法是错误的 因为程序执行是由上到下，从左到右的顺序， 如果这样写，decltype还不知道a，b的类型
  • 任何类型都有默认构造函数 T()会构造一个临时变量，同样U()也会，这样decltype就可以知道返回什么类型了

  • 😄但是谁说就一定会有默认构造函数，可能有些自定义类型没有默认构造函数，那么同样的这样写也会报错，所以这种方法不可取

    • 解决③：返回值后置

      • C++11支持的

auto func(int a, int b) -> decltype(a + b) {   //返回值后置
      return a + b;
}
  • 这样就解决了上面所有问题

用模板实现Vector

  • ❓为什么这里的Vector默认构造函数开辟空间不用new而用malloc呢

    • 因为T类型可能没有默认构造函数，用new无法初始化

  • 在模板类中实现拷贝构造函数——必须用原地构造

  • size成员方法必须定义成const修饰的成员方法——为了访问被const修饰的vector

  • push_back也要实现两个，一个传入左值，一个传入右值

    • 都采用原地构造的形式加入元素

    • 实现右值的push_back()，要明确调用的是T类型的移动构造

      • 35行，右值形参不能绑定在左值引用上

    • 解决：👇

      • 将swap参数类型改为T &&,此时并不表示参数类型是T类型的右值引用，这里的T &&表示我将要传入一个引用（包括左值引用和右值引用）
      • ❓此时问题又来了👉24行代码报错了👇
      • ⭐至此涉及到了一个新的概念：引用折叠
        • 我们想要将一个左值(n)和另一个左值(m)交换，此时模板会推导我们传进去的是右值还是左值，若是左值则T的类型变为int &
        • 也就是说23行这行代码把T展开后是这样的👉void swap(int &&&a, int &&&b)
        • 这就是引用折叠，其实在模板中编译器规定，出现奇数个&则代表传进去的参数是一个左值，若是偶数个&则为右值
        • ❗所以此时T的类型为一个左值引用，定义一个左值引用必须初始化，所以24行代码才会报错

      • 所以这就是模板的难学之处（变态之处），若是不知道引用折叠，这段代码就是极其难找的BUG❗

模板的偏特化和特化

  • 模板的偏特化：在某些特殊情况下，模板的行为方式有些不太一样
    • 可以看成相应模板的重载形式
    • 用来处理当前类型是指针或者数组这些情况的
  • 模板的特化：模板的性质都没有了，模板类型全都实例化完了，但是本质上还是一个模板
    • 可以理解为字面量，不可扩展的模板

  • 为什么模板的特化要加上template<>

    • 因为我们想要显示的调用模板时，希望能准确的匹配到这类模板，否则会匹配到其它形式的模板, 而不是你想调用的特化模板函数

可变参数模板

  • 用变参模板打印若干个参数
    • ARGS...表示参数包
      • 上面的参数args前面有省略号，所以它就是一个可变模版参数，我们把带省略号的参数称为“参数包”，它里面包含了0到N（N>=0）个模版参数。我们无法直接获取参数包args中的每个参数的，只能通过展开参数包的方式来获取参数包中的每个参数，这是使用可变模版参数的一个主要特点，也是最大的难点，即如何展开可变模版参数。
    • 两种方式可以展开可变参数模版函数
      • 用递归函数
      • 用逗号表达式
    • 可变参数模板类
      • 展开一般需要定义两到三个类，包括类声明和偏特化的模板类。
