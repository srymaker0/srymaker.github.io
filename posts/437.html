操作系统

什么是应用程序？

  • 可执行的文件（程序的二进制代码和数据）和其他数据文件

    • Linux支持多种可执行文件格式
    • ELF（Executable Linkable Format）是其中最常用的

  • 运行的程序称为进（正在运行的）程（程序）

    • 操作系统中有很多进程对象
    • 在运行时，进程会
      • 在CPU上执行，进行计算
      • 使用操作系用API访问操作系统中的其他对象

系统中常用的应用程序

Core Utilities（coreutils）

  • standard programs for text and file manipulation
  • 系统中安装的时GNU Coreutils

系统/工具程序

  • bash, apt, ip, ssh, vim, tmux, jdk, python, …

  • Ubuntu Packages —— 支持文件名检索；因此缺少什么东西这是最佳STFW的入口

其他各种应用程序

  • 浏览器、音乐播放器…..

ELF二进制文件

首先，可执行文件也是普通的文件

  • 操作系统里的一个对象

  • 一个存储在文件系统（通常是存储设备）上的字节序列

    • 与大家平时创建的文本文件没有本质区别

    • 操作系统提供API打开、读取、改写（都需要相应的权限）

    • 因此我们可以用vim，cat，xxd等命令查看可执行文件

使用file命令 查看可执行文件a.out

☝ELF

正常打开ELF文件查看如下：

  • 可以看到—color等字符串常量存在ELF文件中

使用readelf解析ELF文件，主要关注：

  • ELF文件的header(元数据)——文件内容的分布，指令集体系结构，入口地址等等
  • ELF的program headers——决定ELF应该如何被加载器加载（执行）
    • /usr/include/elf.h

没有magic，应用程序就是一个普通的文件，按照特定的格式（ELF）存储在磁盘上，通过操作系统加载，运行

应用程序怎么和操作系统交互

尝试1:在操作系统上写一个“最小”的Hello World

#include <stdio.h>
int main() {
    printf("Hello World!\n");
    // c标准默认返回0
}

我们使用gcc 编译后生成hello.o文件，再用命令ld链接惊奇的发现致命问题❗

  • 程序出现未定义错误，为什么是puts，而不是printf？

    • gcc在 -O0选项下依然会进行一定程度的编译优化

    • puts是库函数（libc），需要进行链接，若链接进来就不是“最小”的Hello World了

  • 找不到入口函数 _start？

    • _start是连接器默认的入口
    • 可以用-e指定，比如ld -e main printf（被优化成的puts）是库函数的一部分

尝试3：使用汇编

.globl foo
foo:
    movl $1,    %eax # write (
    movl $1,    %edi #    fd=1,
    movq $s,    %rsi #    buf=s,
    movl $(e-s),%edx #    count=e-s
    syscall          # );

    movl $60,   %eax # exit(
    movl $1,    %edi # status=1;
    syscall   		 # );	

s:
    .ascii "\033[01;31mHello, OS World\033[0m\n"
e:

代码理解：使用了两个系统调用——write，exit

  • write系统调用向文件描述符为1的文件写入了大小为e-s的字符串常量s
  • exit退出程序，并设置1

进入gdb：

  • 可以看到程序经过指令syscall成功在终端打印出了Hello，OS World，❗这个syscall可能有上千条指令

  • 可见我们运行如此简单的程序，其实在操作系统层面已经经过了非常久的时间

⭐回答应用程序（Hello World）怎么和操作系统交互这个问题

​ ——通过设置正确的参数和syscall指令去调用操作系统的API

❓问题：syscall到底在哪

代码：

#include <unistd.h>
#include <sys/syscall.h>

#define LENGTH(arr) (sizeof(arr) / sizeof(arr[0]))

const char hello[] = "\033[01;31mHello, OS World\033[0m\n";

int main() {
    syscall(SYS_write, 1, hello, LENGTH(hello));
    syscall(SYS_exit, 1);
}

使用objdump查看：阅读man手册

[-d]：打印汇编指令

使用以下命令显示源文件函数汇编指令

gcc -g syscall-demo.c
objdump -S a.out
