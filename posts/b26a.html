auto

  • 作用：在C++中可以自动推导变量的类型

    • 其实在C语言中就有，只不过在C语言中在变量类型前加auto表现的就像局部变量，多余
    • ❗在编译期的行为

  • 限制

    • ①不能作为函数参数
    • ②不能作为模板参数（C++11）, 但是在C++14和C++17中可以作为模板参数
    • ③不能定义数组
    • ④不能定义非静态的成员属性

  • 容器

    • 封装好的数据结构——比如：map<int, int>

    • 遍历容器的工具类型：迭代器类型——map<int, int>::iterator👉遍历整型到整型的map类型的迭代器类型

    • iter->first👉map中的key值， iter->second👉map中的value值​

      map<int, int> arr;   
  for (int i = 0; i < 10; i++) {
      arr[rand() % 100] = rand();
  }
  map<int, int>::iterator  iter = arr.begin();       //iter是一个变量
  while (iter != arr.end()) {
      cout << iter->first << " " << iter->second << endl;
      iter++;
  }
for (auto x : arr) {    //for循环的特殊形式
      cout << x.first << " " << x.second << endl;
  }

    • 但是这个迭代器类型太长了，所以auto关键字就起作用了，可以把第5行代码改成下面这种形式

      auto iter = arr.begin();        //auto会自动推导iter的类型  iter的类型由arr.begin()的返回值类型决定

  • 问题

    auto y = 12.3;      //编译器怎么推导出y到底是double类型还是float类型

//使用typeid运算符， 返回的是类型信息对象，若要获取类型信息的名字， 就像下面这样写
cout << typeid(y).name() << endl;       //输出一个d，表示double类型

    • typeid()

      • 每一种类型在typeid中的哈希值都是不一样的，可以通过比较哈希值来判断一个变量到底是不是这种类型的

      • 在模板中功能更大

auto y = 12.3;  
  	if (typeid(y).hash_code() == typeid(float).hash_code()) {
          cout << "float type" << endl;
      }
      if (typeid(y).hash_code() == typeid(double).hash_code()) {
          cout << "double type" << endl;
      }

constexpr——表达式常量

  • 区别于const
关键字 发生时段 定义的常量值 语义
const 运行期 可以在运行期确定 不严谨
constexpr 编译期 必须在编译期就确定 严谨 
int main() {	
	const int x = 123;
    constexpr int y = 123;
    *const_cast<int *>(&x) = 456;       //const_cast可以修改const变量
    *const_cast<int *>(&y) = 456;
    cout << &x << endl;
    cout << &y << endl;
    cout << *const_cast<int *>(&x) = 456;
    cout << *const_cast<int *>(&y) = 456;
    cout << x << endl;      // 输出123
    cout << *(&x) << endl;   // 输出456
    cout << *(&y) << endl;   // 456
}

  • 第10行为什么会输出123？

    • 这就是常量折叠. 这个”常量折叠”是 就是在编译器进行语法分析的时候，将常量表达式计算求值，并用求得的值来替换表达式，放入常量表。可以算作一种编译优化。因为编译器在优化的过程中，会把碰见的const全部以内容替换掉（跟宏似的: #define pi 3.1415,用到pi时就用3.1415代替），这个出现在预编译阶段；但是在运行阶段，它的内存里存的东西确实改变了!!! 简单的说就是，当编译器处理const的时候，编译器会将其变成一个立即数。

    • 因为编译器认为x是一个常量，作了优化，可以理解为直接从第2行代码跳到了第10行

    • 取地址是一个运行期的功能，不会被编译器优化，所以能改变x的值
constexpr int f(int x) {
    return 2 * x;
}

class A {
public :
    constexpr A (int x) {}    //说明是编译期常量方法
};

int main() {
    int n;
    cin >> n;
    const int x = n + 123;           //正确
	constexpr int y1 = n + 123;       //报错, 运行期才能确定
	constexpr int y2 = f(123);        //若f函数的返回值类型前不加constexpr修饰 这行代码会报错
	const A a(123);
    constexpr A b(123);      
}
  • constexpr所定义的变量，在编译期就要确定变量的值是多少。
  • 同样的y变量若是接受一个函数返回值，那么这个函数的返回值也必须修饰为constexpr，让编译器知道这是在编译期就能够确定的值
  • constexpr定义的对象调用的构造函数也要用constexpr修饰

nullptr

#include <iostream>
#include <string>
using namespace std;

void func(int x) {
    cout << __PRETTY_FUNCTION__ << endl;           //用于显示调用的函数名
    cout << x << endl;
    return ;
}

void func(int *x) {
    cout << __PRETTY_FUNCTION__ << endl;           
    cout << x << endl;
    return ;
}

int main() {
    //cout << NULL << endl;            //输出0
    //cout << nullptr << endl;         //输出nullptr

    if (NULL) {
        cout << "true" << endl;
    } else {
        cout << "false" << endl;
    }
    if (nullptr) {
        cout << "true" << endl;
    } else {
        cout << "false" << endl;
    }
    func(nullptr);
    //func(NULL);                   //报错
    return 0;
}
  • NULL和nullptr
    • NULL是一个整数0，同时也代表了空地址? 32行不知道调用哪个func
    • ⭐nullptr只代表了空地址，更加明确，建议在C++中若想表示空地址，尽量用nullptr

左值与右值

#include <iostream>
using namespace std;

#define func(x) __func(x, "func(" #x ")")
#define func2(x) __func2(x, "func2(" #x ")")
void __func2(int &x, const char *str) {
    cout << str << " is left value" << endl;
    return ;
}

void __func2(int &&x, const char *str) {
    cout << str << " is right value " << endl;
    return ;

}
void __func(int &x, const char *str) {    //左值引用
    cout << str << " is left value" << endl;
    func2(x);
    return ;
}

void __func(int &&x, const char *str) {   //右值引用定义形式
    cout << str << " is right value " << endl;
    //func2(x);         			//x就是一个变量，此时x表现的是左值特性  
    //func2(move(x));  				//强制变右值特性   
    func2(forward<int &&>(x));      //准确的向前传递x这个右值    
    return ;
}

int main() {
    int x = 1234, y = 456;
    func(1234);       //右值
    func(x);          //到了下一行我们通过x就能访问到x的值
    func(x + y);      //x + y存储在一个临时变量当中，代码下一行无法通过单一变量访问到
    func(x++);        //x++返回的是x存储的数据，是一个临时变量，并且是一个数据值，
    func(++x);        //直接对x本身操作，返回的是x
    func(x + 123);    //表达式作用在临时变量中
    cout << x << endl;   
    func(x *= 2);     //直接对x作用，返回的也是x
    cout << x << endl;
    func(y += 3);
    cout << y << endl;
    func(y * 3);
    cout << y << endl;
    return 0;  
}

  • 变量x是一个左值，绑定在左值上的引用就是左值引用
    • 语法：int y = 123; int &x = y; 此时x就是一个左值引用
  • 常量1234是一个右值， 绑定在右值上的引用就是右值引用
    • 语法 ：int &&x = 123; 此时x就是一个右值引用
  • ❓如何区别左值和右值
    • 当我们看到一个值时，到了代码的下一行的时候，能通过单一变量(感觉就是作用在本身)访问到相关的值，能->左值 不能->右值
    • 常量一定是一个右值
    • ❗注意：左值，左值引用，右值，右值引用，这四个东西其实并不冲突，可以当成四种概念去学！
  • move和forward
    • 作用：都可以把一个具有左/右值特性的变量强制转换为右/左值特性
    • 区别：forward更加灵活，可以更改类型，forward (x) 表示一定会匹配第一个参数是int &&的函数 👉 参数的完美转发
    • 准确的调用到函数的某种重载形式
  • stackoverflow中对move问题的回答What is move semantics?
