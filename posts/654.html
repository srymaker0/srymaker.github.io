746. 使用最小花费爬楼梯

class Solution { 
public:
	int minCost(vector<vector<int>>& costs) { 
        int n = costs.size(); 
        vector<vector<int>> dp; 
        for (int i = 0; i < 2; i++) dp.push_back(vector<int>(3));
		for (int i = 0; i < 3; i++) dp[0][i] = costs[0][i];
		for (int i = 1; i < n; i++) {
            // 滚动数组
			int ind = i % 2;
			int pre_ind = !ind;
			dp[ind][0] = min(dp[pre_ind][1], dp[pre_ind][2]) + costs[i][0];
			dp[ind][1] = min(dp[pre_ind][0], dp[pre_ind][2]) + costs[i][1];
			dp[ind][2] = min(dp[pre_ind][0], dp[pre_ind][1]) + costs[i][2];
		}

		int ind = (n - 1) % 2;
		return min(dp[ind][0], min(dp[ind][1], dp[ind][2]));
	}
};
  • $dp[n][i]$：n个房子，最后一个房子涂i种颜色的最少花费
  • $dp[n][i] = min(dp[n-1][剩余两种颜色]) + costs[n][i]$

198. 打家劫舍

  • 状态定义：$dp[n][0/1]$ : 当到第n家时，偷还是不偷所达到的最大收益
    • $dp[n][0] = max(dp[n - 1][0], d[n - 1][1])$➡️若当前不偷，则最大收益为前一家不偷/偷的最大值
    • $dp[n][1] = dp[n - 1][0] + nums[i]$➡️若当前偷，则最大收益为前一家不偷+当前的价值

  • $dp[i]=max(dp[i-1]*nums[i], nums[i])$

322. 零钱兑换

  • 状态定义——$dp[n]$：以每一张面额作为最后一张面额所凑得n的最少硬币个数
    • $dp[n]=min(dp[n-v_i]+1)$——$v_i$：第i张硬币的面值，$dp[n-v_i]$：凑够$n-v_i$需要的最少硬币个数最后还要加上当前的硬币值$v_i$
    • $n-v_i$需要合法，也就是能够拼凑出

2：状态定义👇

正向刷表：当前$dp[i][j]$依赖上一行的$dp[i-1][j]$和这一行的$dp[i][j-x]$

反向刷表：当前$dp[i][j]$依赖上一行的$dp[i-1][j]$和上一行的$dp[i-1][j-x]$

300. 最长递增子序列

  • 状态定义——$dp[n]$：以n位置为结尾的最长递增子序列的长度
    • $dp[n]=max(dp[j]+1)|j<n,val_j<val_n$

714. 买卖股票的最佳时机含手续费

状态定义：$dp[i][0/1]$——第i天持有股票(1)或者不持有股票(0)所获得的最大收益

状态转移：

  • $dp[i][0]= max\left\{\begin{aligned}&dp[i-1][0]&前一天也不持有，第i天也不买入\\&dp[i-1][1]+prices[i]-fee&前一天持有，第i天刚好卖了\end{aligned}\right.$

  • $dp[i][1]= max\left\{\begin{aligned}&dp[i-1][1]&第i天不卖且前一天持有\\&dp[i-1][0]-prices[i]&前一天卖了，第i天买入\end{aligned}\right.$

213. 打家劫舍 II

注意——房屋成环了

先考虑不成环的情况，最后分别求第1号房和最后一间房偷或者不偷的最大值

状态定义：$dp[n][0/1]$ : 当到第n家时，偷还是不偷所达到的最大收益

  • $dp[n][0] = max(dp[n - 1][0], d[n - 1][1])$➡️若当前不偷，则最大收益为前一家不偷/偷的最大值
  • $dp[n][1] = dp[n - 1][0] + nums[i]$➡️若当前偷，则最大收益为前一家不偷+当前的价值

416. 分割等和子集

思路：两个子集要想等，先将nums数组求和再除2，求能否凑出此数字

状态定义：

  • $dp[i][j]$:前i个数是否能凑出j值
  • $dp[i][j] = dp[i - 1][j] | dp[i - 1][j - num[i]]$，只要有一个能凑出即可

可达数组：能够达到某种状态的数组

尽管代码实现中是一维数组的实现，但是在思维逻辑层面还是2维的。

$f[j]$有隐含的i条件，状态转移代码要保证等式右边的$f[j]$、$f[j-x]$在【前i个数】维度的索引为$i - 1$；如果正序，$f[j-x]$对应的索引已经变为 i 了，而$f[i-1][j-x]$已经丢失

❗ 这里j的遍历范围变成了$sum\to x$

  • 这是因为没遍历到的$x\to 1$部分，其值不需要改变，保持即可 [f是一维的]
  • 只需要关注$j-x \to j$这段范围即可

474. 一和零

思路：本质上还是0/1背包问题——m，n是资源限制条件，把每个字符串想象成物品，而每个物品的价值可以想象成1

状态定义

  • $dp[i][m][n]$:取前i个字符串，在最多有m个0，n个1的条件下所能取得的最大子集
  • $dp[i][m][n] = max(dp[i][m][n], dp[i][m - cnt0][n - cnt1] + 1)$

状态转移：

518. 零钱兑换 II

状态定义

  • $f[i][j]$：使用前i种钱币凑出j元的方法总数
  • $f[i][j]=f[i-1][j]+f[i][j-x]$

❗注意：每个数字可以选无数次，不管选了多少个第 i 种物品，物品种数还是前 i 种

优化：正向刷表

377. 组合总和 Ⅳ

状态定义：

  • $dp[n]$:用所有硬币拼凑n元钱的方法总数
  • $dp[n]=\sum{dp[n-x_i]}$ $1 \le i \le n$

注意：这样求解时不会考虑顺序问题
