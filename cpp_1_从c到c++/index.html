从c到c++

了解C++

  • 主要学习c++11标准
    • 11标准比03标准更新了很多， 而14，17标准变化不大，20标准太新，应用场景受局限

  • C++可以认为是C语言的超集

    • C++继承了C语言绝大多数的语法规则以及关键字部分
      • 在语言的发展过程中C语言有一些语法规则不太适用，所以被C++给删除了——这部分被删除的不在我们的知识体系内
      • 所以可以认为我们所掌握的c语言所有的知识都被c++继承下来了
    • :x:cin和cout属于C++， scanf 和printf属于C语言
      • C++继承了C， 所以这种说法是不对的， 继承了的东西就已经归自己了

  • C++继承了C语言29个标准头文件

    • 标准头文件：C语言官方固有的，在多少标准中被引入的
    • 非标准头文件：在特定编程环境中特有的头文件，比如<windowns.h>是windowns中才有的
      • 在实际的工程开发中：是标准头文件和非标准头文件混合编程

  • 语言联邦（特性）——C语言 + 模板 （泛型编程）+ Lambda表达式（函数式编程） + 类和对象

    • :bangbang:对于所有C++中的语言特性我们都需要精通吗？

      • 编程范式的本质作用：提高开发效率，找bug效率，后期维护的效率

      • 我们在完成1个程序的时候可能只需要用到1种或者2种编程范式，大部分用到的是面向过程和面向对象——那么我们为什么学习其它的编程范式呢？:point_down:

      • 学习模板和Lambda可以提高我们的编程效率

      • 船长结论：当相关的编程范式可以提高我们的开发效率的时候，可以考虑换一种编程范式

      • 【ps】很多C++设计的观点是仁者见仁，智者见智的

  • 异常处理：挽回程序崩溃退出的一种方式

    • 在某些运行时错误不会导致程序的崩溃

如何把C++学成一门简单的语言

  • 日后我们学习其它编程语言的时候我们应该如何学习
    • 一句话——按照编程范式分门别类的学习
    • 当你把一种语言特性和一种编程范式一一对应的时候 这个语言学起来就会很简单
  • 推荐书籍
    • C++ primer plus / C++ primer
    • Effective C++
    • C++对象模型

Effective C++ 读书分享

条款03：尽可能使用const

  • 将某些东西声明为const可以帮助编译器检测除错误用法
  • 编译器强制实施 bitwise constness（数据意义的const），但你编写程序时应该使用概念上的常量性 conceptual constness （逻辑上的const）
    • 对于编译器来说想实现真正数据意义上的const是很难的👉比如说指针，定义一个常量指针，我们认为地址和指向的内容都不允许修改，但实际上只有地址是不能修改的，指针所指向的内容我们可以修改；
    • 逻辑上的const：对于用户来说$A$对象中的$X$才是最重要的数据，我们定义的$y$成员只是为了完成一种用户不可知的需求，并不改变$X$的值 对于用户来说并不影响， 这就是逻辑上的const。
    • :bangbang:代码29行会报错: 原因是say2()方法可能会修改const类型对象a的成员属性。
  • 当const和non-const成员函数有着实质等价的实现时， 令non-const版本调用const版本可以避免代码重复

条款04：确定对象被使用前已经先被初始化

  • 为内置型对象进行手工初始化，因为C++并不保证会初始化它们
  • 构造函数最好使用成员初值列，而不要再构造函数本体内使用赋值操作 - 初始化列表：$A() : x(123), y(0)$ 冒号后面的就是初始化列表，用于初始化成员属性 - 对于初始化内置类型来说 调用构造函数和赋值运算符进行初始化都可以 两者无差别 - 但是对于一个类类型来说，调用赋值运算符初始化和调用构造函数初始化两者截然不同，初始化类需用构造函数
  • 初值列列出的成员变量，其排列次序该和它们在class中的声明次序相同
    • 实际顺序 = 代码书写顺序
    • 22行就算写成$A() : y(0), x(123)$ 也是$x$先被初始化， 因为x先被定义，编译器的魅力:exclamation:。
  • 为免除“跨编译单元之初始化次序”问题，请以local static对象替换non-local static对象
